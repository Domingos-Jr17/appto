# Rules Distill

Scan installed skills, extract cross-cutting principles, and distill them into rules.

## Process

1. Read all installed skills
2. Identify common principles across skills
3. Extract cross-cutting rules (testing, security, coding style, git)
4. Propose new rule files or updates to existing rules
5. Ask user to confirm before writing

## Output

Proposed rule files in `.ai/rules/` or updates to existing rules in `catalog/rules/`.
